# Community Guidelines

## Mission Alignment

ChittSaathi exists to support student mental health. Contributions should improve safety, accessibility, reliability, and empathy.

## Communication Principles

- Be kind, direct, and respectful.
- Critique ideas, not people.
- Keep conversations actionable and specific.

## Product and Data Principles

- Prioritize user privacy and data minimization.
- Avoid features that could encourage self-harm, harassment, or stigma.
- Design for low-bandwidth and mobile-first contexts.
- Use inclusive language suitable for diverse student audiences.

## Contribution Principles

- Start with an issue for large changes.
- Keep pull requests scoped to one clear purpose.
- Add or update docs when behavior changes.
- Add tests when practical, especially for backend APIs.

## Maintainer Workflow Guidelines

- Label issues clearly (`bug`, `enhancement`, `good first issue`, `help wanted`).
- Ask for clarification early when reports are incomplete.
- Provide constructive review feedback and clear merge criteria.

## Moderation

Moderation decisions follow `CODE_OF_CONDUCT.md` and are made to protect community safety.
